# Review Rubric

## Review Criteria by Artifact Type

### Research Plan

| Criterion | Description | Questions to Ask |
|-----------|-------------|-------------------|
| Clarity | Is the plan clear and well-structured? | Can someone else understand and follow this plan? Is the writing unambiguous? |
| Feasibility | Can this plan realistically be executed? | Are the resources, timeline, and methodology realistic? Are dependencies identified? |
| Originality | Does the plan address a novel question? | Does this fill a gap in existing literature? Is the approach innovative? |
| Significance | Would the results matter? | Who would benefit from this research? Does it advance the field? |

### Literature Review

| Criterion | Description | Questions to Ask |
|-----------|-------------|-------------------|
| Relevance | Are the selected papers relevant to the research question? | Does each paper directly relate to the plan? Are there gaps in coverage? |
| Accuracy | Are the summaries correct? | Do the key contribution and methodology sections accurately reflect the paper? Are claims supported? |
| Completeness | Are all important works included? | Are seminal papers covered? Are recent advances included? Are opposing viewpoints represented? |
| Coverage | Is the breadth of literature adequate? | Are all relevant subfields covered? Is the time range appropriate? Are all key entities and concepts extracted? |

### Experiment Design

| Criterion | Description | Questions to Ask |
|-----------|-------------|-------------------|
| Validity | Will the experiment actually test the hypothesis? | Do the variables relate to the hypothesis? Are controls adequate? Could results have alternative explanations? |
| Reliability | Will the experiment produce consistent results? | Is the procedure reproducible? Are measurements well-defined? Is the data collection plan clear? |
| Reproducibility | Can someone else reproduce this experiment? | Are all parameters specified? Is the procedure step-by-step? Are tools and datasets identified? |
| Ethicality | Are there ethical concerns? | Does this involve human subjects or sensitive data? Are potential harms identified and mitigated? |

### Paper Draft

| Criterion | Description | Questions to Ask |
|-----------|-------------|-------------------|
| Structure | Is the paper well-organized? | Does it follow academic conventions? Are sections logically ordered? Is the narrative coherent? |
| Clarity | Is the writing clear? | Are arguments easy to follow? Is jargon explained? Are figures and tables effective? |
| Evidence | Are claims supported by evidence? | Are results presented clearly? Are statistical tests appropriate? Do conclusions follow from data? |
| Citation completeness | Are all sources properly cited? | Are all referenced works in the wiki? Are citations complete and accurate? Is the bibliography consistent? |

---

## Severity Levels

| Severity | Definition | Action |
|----------|------------|--------|
| Critical | Blocks progress or invalidates the artifact | Must be addressed before proceeding |
| Major | Significant issue that should be fixed | Should be addressed before next phase |
| Minor | Small improvement or informational | Can be addressed at any time |

---

## Review Process

1. **Read the artifact** — Read the full artifact (plan, lit review, experiment design, or paper draft) from the wiki.

2. **Apply the relevant rubric** — Use the criteria table for the artifact type. Evaluate each criterion.

3. **Classify issues** — For each issue found, assign a severity level:
   - **Critical**: Blocks progress or makes the artifact invalid. Examples: hypothesis is untestable, methodology is fundamentally flawed, key papers are missing.
   - **Major**: Significant issue that should be fixed but does not block all progress. Examples: unclear writing, incomplete coverage, missing controls.
   - **Minor**: Small improvement or informational note. Examples: formatting inconsistency, additional reference suggestion, minor clarity improvement.

4. **Write the review** — Present structured feedback using this format:
   ```
   ## Review: {Artifact Name}
   **Artifact Type**: {Research Plan | Literature Review | Experiment Design | Paper Draft}
   **Date**: YYYY-MM-DD

   ### Summary
   {2-3 sentence overall assessment}

   ### Critical Issues
   {numbered list of critical issues, or "None" if none found}

   ### Major Issues
   {numbered list of major issues, or "None" if none found}

   ### Minor Issues
   {numbered list of minor issues, or "None" if none found}

   ### Strengths
   {What the artifact does well}

   ### Suggested Fixes
   {For each issue, a concrete suggestion for how to fix it}
   ```

5. **Suggest fixes** — For every issue identified, provide a concrete, actionable suggestion. Reference wikilinks where relevant.
